# FILE SYSTEM SECURITY AND AUDITING

The file system of OS stores important or needed data for the user.

1. Password files
2. Previliged data
3. Program and .exe files
4. Databases

A. Files Should have correct premisions
- Who can read it.
- who can modify it.
- who can execute.

B. Sensitive Files should be protected.
1. Password files
2. Configuration files
3. Database backups

C. Only Admins can Modify Critical Settings.
1. Setting should have Proper Description.
2. System Settings should be accessed after displaying the warining applet.
   
D.Logs Tracking:
1. Tracking Who accessed the file.
2. When it occurred.
3. Whether it was successful or failed.
4. Number of attempts made.
This will help us to detect attacks: Insider adn Investigation.
also it detects suspicious activites.
